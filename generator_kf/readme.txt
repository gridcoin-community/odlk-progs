1) generator_kf.exe — это Генератор канонических форм ДЛК10.

В файл config.txt запишите два числа, первое — номер линейки,
второе — сколько КФ сгенерировать.

В файл start.txt запишите стартовые данные — от 0 до 100 элементов ДЛК.
Этот файл не обязателен, и не обязательно должен быть корректным (в том
числе, побочная диагональ может не соответствовать номеру линейки).
Программа сама выберет максимальную корректную стартовую последовательность.
Если стартовые данные — корректная КФ, то вывод начнётся со следующей
в лексикографическом порядке КФ.


2) generator_kf_s.exe — это Генератор случайных КФ ДЛК10.

Файл config.txt аналогичный, файл start.txt не используется.

Вначале программа ищет для заданной линейки все варианты заполнения
первых 10 пустых ячеек, затем случайно выбирает один из них и ищет
все варианты заполнения следующих 10 пустых ячеек, и, также случайно,
выбирает один из этих вариантов. После чего приступает к сплошному
поиску пока не будет найдено нужное число КФ или не будут исчерпаны
все возможные продолжения.

Обе программы выводят КФ-ы в лексикографическом порядке в файл output.txt.


3) generator_kf_odlk.exe — это Генератор КФ марьяжных ДЛК10 (ОДЛК10).

В файл config.txt запишите номер линейки. А в файл start.txt — стартовые
данные, аналогичные стартовым данным программы Генератор КФ. Фактически
это и есть Генератор КФ к которому добавлена проверка КФ но марьяжность.
Программа будет работать пока не найдёт КФ ОДЛК (или не исчерпает линейку).
Найденная КФ ОДЛК будет добавлена к файлу output.txt, записана в файл
start.txt (для продолжения поиска с этой КФ ОДЛК), и выведена на консоль
с вопросом о продолжении работы. Если программу запустить с ключём /m, то
КФ ОДЛК на консоль выводиться не будет и программа продолжит работу без
лишних вопросов.
